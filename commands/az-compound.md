# Command: az-compound

The knowledge management and compounding phase.

**[SA] Time to get smarter!** I'm calling in @scrum-master and @qa.

## 1. Learning Extraction (Via [QA] Argus)

- **[QA] Argus** identifies:
  - **Successful Patterns**: What worked well in the current sprint.
  - **Pitfalls Avoided**: What common mistakes did we sidestep?
  - **Tech Debt Paid**: What code did we simplify?

## 2. Knowledge Summarization (Via [SA] Bob)

- **[SA] Bob** creates a "Sprint Learning Record":
  - Summarizes the "Lessons Learned" from the current sprint.
  - Updates `shared/memory.md` with new, hardened patterns.
  - Moves outdated patterns to a `shared/archive/` if necessary.

## 3. Knowledge Injection (Via [SA] Bob)

- **[SA] Bob** prepares the next sprint:
  - Automatically identifies how current learnings apply to the next task.
  - Alerts the team to new, preferred patterns.
  - Optimizes the next plan based on these efficiency gains.

---

**[SA] Knowledge compounded.** Each sprint makes us faster and more precise. Should we proceed to the retrospective or start the next sprint?
