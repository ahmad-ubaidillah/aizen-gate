# Skill: User Experience Documentation (Guides)

## Overview

Documentation for humans who use the software. Focuses on **Use Cases** rather than implementation.

## Guidelines

- **Step-by-Step**: Use ordered lists with expected outcomes.
- **Visual Context**: Describe what the user should see in the UI/CLI.
- **Troubleshooting**: Provide solutions for known edge cases.
- **Premium Vibe**: Use encouraging, high-end editorial tone.
