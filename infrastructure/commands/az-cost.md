# Command: az-cost

The token optimization and cost analytics phase.

**[SA] Time to save some cash!** I'm calling in @analyst and @devops.

## 1. Token Usage Analysis (Via [ANALYST] Sigma)

- **[ANALYST] Sigma** will analyze:
  - **Consumption**: Monthly, weekly, and daily token trends.
  - **Heaviest Tasks**: Identifying which processes consume the most tokens.
  - **Token-to-Performance Ratio**: Correlating cost with progress on the board.

## 2. API Optimization (Via [DEVOPS] Stark)

- **[DEVOPS] Stark** will recommend:
  - **Filtering**: Using `rtk`-style output filtering to omit noise.
  - **Memory Selection**: Moving long-lived context into compressed memory.
  - **Model Profiles**: Suggesting budget vs. quality models for specific tasks.

## 3. Cost-Saving Implementation (Via [SA] Bob)

- **[SA] Bob** will implement:
  - **Context Pruning**: Selectively removing older history from active memory.
  - **Fact Extraction**: Moving key insights into `shared/memory.md` to avoid re-reading.
  - **Cache Management**: Identifying repeatable patterns that don't need re-generation.

## 4. Cost Reporting (Via [SA] Bob)

- **[SA] Bob** will generate a report:
  - **Estimated Costs**: Based on current usage vs. project scope.
  - **Savings**: Amount saved through optimizations.
  - **Recommendations**: For reducing costs on the next sprint.
